# Ninja Labs NFT 项目需求文档

## 1. 项目概述

### 1.1 项目背景

Ninja Labs 是 Injective (INJ) 区块链的开发者社区，致力于推动 Web3 开发创新和社区协作。该 NFT 项目旨在通过区块链技术构建一个专属的社区身份系统，帮助塑造社区共识，并导向潮流开发社群的氛围。作为一个赛博朋克风格的 NFT 系列，该项目将忍者形象与现代时尚元素结合，激励成员通过贡献提升社区参与度。

### 1.2 项目名称

Ninja Labs NFT

### 1.3 项目目标

- **塑造社区共识**：通过 NFT 作为社区成员的数字身份凭证，强化成员归属感和共识，形成一个活跃、协作的开发者社群。
- **导向潮流开发社群**：以年轻、时尚的赛博潮流忍者设计，吸引年轻开发者加入，推动 INJ 生态的创新和潮流文化。
- **激励社区贡献**：基于贡献度分级系统，鼓励成员参与任务、贡献代码、分享知识等活动，提升社区整体活跃度。
- **长期价值**：NFT 不仅作为身份标志，还可扩展为未来社区治理、专属权益（如 DAO 投票权、独家活动访问）的入口。

### 1.4 关键指标（KPIs）

- 社区成员覆盖率：目标覆盖 95% Ninja Labs 活跃成员。
- 升级率：30% 成员在 3 个月内升级到紫色背景，10% 升级到橙色背景。（先不考虑）
- 社区增长：通过 NFT 项目的文化元素，以及后续的赋能，带动新成员加入率。

## 2. 用户群体

### 2.1 目标用户

- **主要用户**：Ninja Labs 社区成员，包括开发者、设计师、贡献者和 INJ 生态爱好者。年龄段主要为 18-35 岁，偏好 Web3、技术创新和潮流文化。
- **次要用户**：潜在社区加入者，如 INJ 持有者、其他区块链开发者，通过 NFT 吸引他们进入社区。
- **用户痛点**：缺乏可视化的贡献认可系统；社区身份难以量化；缺乏激励机制导致参与度低。
- **用户需求**：易于获取的 NFT 身份；基于贡献的升级路径；时尚、独特的数字艺术收藏。

### 2.2 用户旅程

1. **加入社区**：用户通过 Discord、Telegram 或官网注册 Ninja Labs 社区，完成 KYC 或简单验证后，领取白色背景 NFT。（1 个 INJ）
2. **贡献积累**：参与任务（如代码贡献、bug 报告、社区活动，内容输出），积累积分。
3. **升级 NFT**：积分达到阈值时，触发 NFT 升级（背景颜色变化）。
4. **社区互动**：持有 NFT 可参与专属事件、展示身份、交易或展示在钱包中。

## 3. 设计理念

### 3.1 总体理念

- **赛博潮流忍者**：Ninja Labs 宣传图上的忍者形象，分别作为 NFT。后期每个（类）忍者可以分别做成 ip。
- **艺术风格**：生成艺术（Generative Art），每个 NFT 独特，包括随机生成的忍者特征（如发型、服装、武器），但背景颜色固定为三种级别。使用像素艺术或 3D 渲染，融入潮流元素（如街头服饰、LED 灯效）。
- **颜色分级**：
  - **白色背景**：普通贡献者（新成员），象征纯净起点。
  - **紫色背景**：资深贡献者，象征神秘与深度（积分阈值：待定）。
  - **橙色背景**：顶级贡献者，象征活力与领导力（积分阈值：待定）。
- **独特卖点（USP）**：动态升级机制（非静态 NFT），结合社区贡献积分，实现 NFT 的“进化”，增强持有者的情感绑定。

### 3.2 艺术与 Metadata 细节

- **NFT 组成**：
  - **主体**：不同形象的忍者（traits：头部、眼睛、服装、配件等，确保稀有度分布）。
  - **背景**：仅三种颜色，升级时仅变更背景（保持主体不变，以保留用户情感）。
  - **Metadata**：JSON 格式，包含属性如 “Mint Date”等。存储在 IPFS 上，确保去中心化。
- **稀有度**：80% 常见 traits，15% 稀有，5% 传奇。（80 普通+20 西游）
- **分辨率**：1024x1024 像素，支持钱包展示和 AR 滤镜扩展。

## 4. 功能需求

### 4.1 核心功能

1. **NFT Minting**：
   1. 新成员免费 mint 白色背景 NFT（gas fee 自付）。
   2. 基于 INJ 链，使用 ERC-721 （兼容 Cosmos SDK）。
   3. 限量供应：初始 200 个，动态扩展基于社区规模。
2. **积分系统**：
   1. **追踪方式**：Off-chain（Discord bot 或社区 dashboard）结合 on-chain oracle 验证。
   2. **贡献类型**：代码 PR、活动参与、分享教程等，加分量待定。管理员审核或自动化积分。
   3. **阈值**：0-a 分（白）、a-b 分（紫）、b-c 分（橙）。待定。
   4. **积分查询**：用户可通过某种方式查询当前积分和升级进度，可以是社区中链下查询，也可以是链上查询。待定。
3. **NFT 升级机制**：
   1. **动态升级**：积分达标时，用户触发“升级”交易（后台可以直接修改颜色）。
   2. **费用**：低 gas fee，无额外成本，鼓励升级。
   3. **通知**：社区 bot 推送升级提醒，增强社区认同感。
4. **社区集成**：
   1. **权益绑定**：投票权、激励等等。
   2. **展示与分享**：集成 INJ NFT 市场，支持社交分享。
5. **dApp 接口**：
   1. Web 界面：连接钱包，查看/mint/升级 NFT，积分 dashboard。
   2. API：用于社区工具集成（如积分同步）。

### 4.2 附加功能（MVP 后扩展）

- **交易市场**：二级市场，支持 P2P 交易。
- **合作扩展**：与其他 INJ 项目联动，如 staking NFT 获 yield 等。

## 5. 非功能需求

### 5.2 用户体验

- **易用性**：移动友好 dApp，一键 mint/升级。
- **多语言**：支持英语/中文（针对全球开发者）。
- **兼容性**：支持主流钱包

## 6. 经济模型

### 6.1 Minting 与费用

- 初始 mint 免费（社区空投）。
- 升级费用：仅 gas，~0.1 INJ（待定）。
- 收入来源：未来赋能后，收取二级市场 royalty（5%），用于社区基金。

### 6.2 通胀/通缩

- 总供应：动态，根据社区增长 mint 新 NFT。

## 7. 项目时间表（待定）

- **Phase 1: 规划与设计** (1 week)：需求确认、艺术设计、智能合约开发。
- **Phase 2: 开发与测试** (1-2 week)：dApp 构建、积分系统集成、测试网部署。
- **Phase 3: launch** (1-2 week)：主网上线、社区空投、营销。
- **Phase 4: 迭代** (ongoing)：基于反馈添加功能。
